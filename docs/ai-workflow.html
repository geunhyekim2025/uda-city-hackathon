How this was built with AI — and where the AI cross-checked itself

A SUEWS run is only as good as the scrutiny around it. The agent (driving supy + the SUEWS validation tools) was used not to autocomplete answers, but to catch its own mistakes. ← back to the main story

The workflow, in five steps

  1. Ground the agent in the dataset, not guesses. First it read agent_manifest.yml and risk_bridge.md/.py in full, so every later step used the repo's own contract (canonical config, physics lock, hazard definition).
  2. Confirm it runs. Built a Python 3.12 env, installed supy ≥ 2026.6.5 (the manifest's hard requirement), and ran the repo's own smoke test (asserts T2/QH finite) before any analysis.
  3. Present → risk bridge for all 10 neighbourhoods, using the reference build_risk verbatim, then read the result against the urban form.
  4. Stress the result: humid-heat metrics, a +2.5 K future, and a cool-roof + greening intervention — each validated, each cross-checked.
  5. Deliver: a one-command reproducibility script and this public story site, with an honest "where it holds / breaks" account.

Prompts that mattered

Where AI + human cross-checking changed the answer

These are the moments the collaboration added real value — catching what a single-pass answer would have got wrong.

✓ Caught a saturating metric

The first humid pass kept the 35 °C threshold on apparent temperature. Cross-check: AT runs ~6 °C hotter than T2 here, so every neighbourhood was "dangerous" ~800 h — the signal collapsed. Fix: threshold each humid metric at its own danger level (wet-bulb > 28 °C). Lesson surfaced: change the metric and threshold together.

✓ Exposed a scoring artefact

After the intervention cut Kampong Lama's dangerous hours 60%, its relative risk index stayed pinned at 1.00 and untreated areas' indices rose without getting hotter. Traced to min–max scaling and made a headline caveat: judge by absolute hazard avoided, not a relative index.

✓ Verified the mechanism

The intervention's cooling was confirmed against the surface energy balance — QN↓ (higher albedo), QE↑ (greening), QH↓ ~25% (less sensible heating of the air) — not just asserted from fewer hot hours.

✓ Physically-consistent humidity

Used the model's own 2 m RH2/Q2 (height-consistent with T2) rather than splicing forcing-level RH onto modelled T2 — and stated the choice so a reviewer can object.

Engineering issues the agent caught and fixed

  • matplotlib.cm.get_cmap removed in the installed version → switched to plt.get_cmap.
  • GitHub Pages "build failed" because it ran Jekyll on hand-written static HTML → added docs/.nojekyll (the correct fix).
  • Editing land cover kept surface fractions summing to 1.0, then ran suews validate (zero errors) before the long run; verified the live site end-to-end after each push.

Contribution to the SUEWS community

Reusable findings shared with the SUEWS Discourse hackathon channel — best practice and a critical suggestion, drawn from the work above.

Best practice

A humid-heat threshold has to move with the metric

Switching T2 → apparent temperature but keeping a 35 °C threshold saturates the hazard (~800 h "dangerous" everywhere) and scrambles the ranking. Pair every metric with a metric-appropriate threshold (e.g. wet-bulb > 28 °C). Swapping the metric alone is a silent failure mode.

Critical suggestion

Relative (min–max) risk indices hide interventions

A min–max index can show a 60%-effective intervention as helping nobody (the worst site stays at 1.0) and can inflate untreated sites. Report the absolute hazard change (hours of dangerous heat avoided) alongside any relative index, and note such indices don't transfer between cities or states.

Best practice

Validate before you run; use the model's own 2 m state

Run suews validate before every long simulation (it catches structure issues up front), and use SUEWS' own RH2/Q2 for humid-heat indices rather than splicing forcing-level RH onto modelled T2.

uv venv --python 3.12 .venv
uv pip install --python .venv "supy>=2026.6.5" pyyaml matplotlib pandas
.venv/bin/python scripts/reproduce.py
